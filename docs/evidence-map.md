# Chorus — Evidence Map

## Purpose

This document maps Chorus's engineering claims to the artefacts that support them.

The map is structured for architecture review: each capability links to the artefacts and code locations that demonstrate it. Phase 1A rows point to implemented happy-path evidence. Phase 1B rows point to shipped governance/failure fixtures. Phase 1C rows package those links for asynchronous review. Phase 2 rows distinguish implemented Phase 2A evidence from later planned milestones.

## Position

Chorus addresses governed runtime orchestration for agent-enabled business processes: durable workflow state, provider/model policy, tool authority, auditability, observability, and evaluation. The Phase 1 evidence is intentionally local to this repository: a reviewer should be able to move from claim to implementation, contract, test, operational surface, or ADR without relying on hidden context.

## How to use this map

Use the matrix from top to bottom when reviewing the architecture narrative. Use the artefact index underneath it when starting from a file, dashboard, or test and working back to the claim it supports. Rows marked Phase 1A point to implemented happy-path evidence. Phase 1B governance/failure rows point to the shipped fixture evidence.
